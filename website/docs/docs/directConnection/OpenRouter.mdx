# OpenRouter

import openRouterLogo from '/img/openRouterLogo.png';

# <img src={openRouterLogo} className="adaptive-logo-filter" width="34" style={{float: 'left', marginRight: '10px', marginTop: '12px'}} /><span className="direct-service-title">OpenRouter</span>

Properties used to connect to [OpenRouter](https://openrouter.ai/).

### `openRouter` {#openRouter}

- Type: `true` | \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `model?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `models?: string[]`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `max_tokens?: number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `max_completion_tokens?: number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `stop?: string | string[]`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `system_prompt?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`tools?: OpenRouterTool[]`](#OpenRouterTool), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`tool_choice?: OpenRouterToolChoice`](#OpenRouterToolChoice), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `parallel_tool_calls?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`response_format?: OpenRouterResponseFormat`](#OpenRouterResponseFormat), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`reasoning?: OpenRouterReasoning`](#OpenRouterReasoning), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`function_handler?: FunctionHandler`](#FunctionHandler) <br />
  \}
- Default: _\{model: "openai/gpt-4o"\}_

Connect to OpenRouter's [Chat Completion API](https://openrouter.ai/docs/api/api-reference/chat/send-chat-completion-request). You can set this property to _true_ or configure it using an object:<br />
`model` is the name of the model to be used by the API (e.g., "openai/gpt-3.5-turbo"). <br />
`models` is an optional list of fallback model identifiers used in order if the primary `model` is unavailable. <br />
`max_tokens` limits the maximum number of tokens in the generated response (1 to model's context length). <br />
`max_completion_tokens` is the newer alternative to `max_tokens` for capping the response length. <br />
`stop` is a string or array of strings at which the model will stop generating further tokens. <br />
`system_prompt` provides behavioral context and instructions to the model. <br />
[`tools`](#OpenRouterTool) defines available function declarations for the model to call. <br />
[`tool_choice`](#OpenRouterToolChoice) controls whether and which tool the model should call. <br />
`parallel_tool_calls` toggles whether the model may request multiple tool calls in parallel (defaults to _true_). <br />
[`response_format`](#OpenRouterResponseFormat) constrains the model output (e.g. JSON or a specific JSON schema). <br />
[`reasoning`](#OpenRouterReasoning) configures reasoning behavior for models that expose a thinking step. <br />
[`function_handler`](#FunctionHandler) enables function calling capabilities for tool use. <br />

## Tool Calling

OpenRouter supports [function calling](https://openrouter.ai/docs/guides/features/tool-calling) functionality:

### `OpenRouterTool` {#OpenRouterTool}

- Type: \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `type: "function"`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `function:` \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `name: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `description: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `parameters: object` <br />
  \}}

Array describing tools that the model may call. <br />
`type` must be "function" for function tools. <br />
`name` is the name of the tool function. <br />
`description` explains what the tool does and when it should be used. <br />
`parameters` defines the parameters the tool accepts in JSON Schema format. <br />

### `OpenRouterToolChoice` {#OpenRouterToolChoice}

- Type: `"auto"` | `"none"` | `"required"` | \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `type: "function"`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `function: {name: string}` <br />
  \}

Controls how the model selects tools. <br />
`"auto"` lets the model decide when to call a tool. <br />
`"none"` prevents the model from calling any tool. <br />
`"required"` forces the model to call at least one tool. <br />
An object form forces the model to call the named function.

### `OpenRouterResponseFormat` {#OpenRouterResponseFormat}

- Type: \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `type: "text" | "json_object" | "json_schema"`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `json_schema?:` \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `name: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `description?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `schema: object`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `strict?: boolean` <br />
  &nbsp;&nbsp;&nbsp;&nbsp; \} <br />
  \}

Constrains the model's response shape. <br />
`type` selects the format: plain text, an arbitrary JSON object, or a value matching a JSON Schema. <br />
`json_schema` is required when `type` is `"json_schema"` and describes the desired shape.

### `OpenRouterReasoning` {#OpenRouterReasoning}

- Type: \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `effort?: "low" | "medium" | "high"`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `max_tokens?: number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `exclude?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `enabled?: boolean` <br />
  \}

Configures reasoning behavior for models that support a separate thinking step. <br />
`effort` is a high-level knob for how much reasoning to perform. <br />
`max_tokens` caps the token budget allocated to reasoning. <br />
`exclude` hides the reasoning tokens from the returned response. <br />
`enabled` toggles reasoning on or off when the model supports it.

### `FunctionHandler` {#FunctionHandler}

- Type: ([`functionsDetails: FunctionsDetails`](/docs/directConnection#FunctionsDetails)) => `{response: string}[]` | `{text: string}`

The actual function that the component will call if the model wants to use tools. <br />
[`functionsDetails`](/docs/directConnection#FunctionsDetails) contains information about what tool functions should be called. <br />
This function should either return an array of JSONs containing a `response` property for each tool function (in the same order as in [`functionsDetails`](/docs/directConnection#FunctionsDetails))
which will feed it back into the model to finalize a response, or return a JSON containing `text` which will immediately display it in the chat.

// using JavaScript for a simplified example

chatElementRef.directConnection = {
  openRouter: {
    tools: [
      {
        type: 'function',
        function: {
          name: 'get_current_weather',
          description: 'Get the current weather in a given location',
          parameters: {
            type: 'object',
            properties: {
              location: {
                type: 'string',
                description: 'The city and state, e.g. San Francisco, CA',
              },
              unit: {type: 'string', enum: ['celsius', 'fahrenheit']},
            },
            required: ['location'],
          },
        },
      },
    ],
    function_handler: (functionsDetails) => {
      return functionsDetails.map((functionDetails) => {
        return {
          response: getCurrentWeather(functionDetails.arguments),
        };
      });
    },
    key: 'placeholder-key',
  },
};
```

// using JavaScript for a simplified example

chatElementRef.directConnection = {
  openRouter: {
    tools: [
      {
        type: 'function',
        function: {
          name: 'get_current_weather',
          description: 'Get the current weather in a given location',
          parameters: {
            type: 'object',
            properties: {
              location: {
                type: 'string',
                description: 'The city and state, e.g. San Francisco, CA',
              },
              unit: {type: 'string', enum: ['celsius', 'fahrenheit']},
            },
            required: ['location'],
          },
        },
      },
    ],
    function_handler: (functionsDetails) => {
      return functionsDetails.map((functionDetails) => {
        return {
          response: getCurrentWeather(functionDetails.arguments),
        };
      });
    },
    key: 'placeholder-key',
  },
};

function getCurrentWeather(location) {
  location = location.toLowerCase();
  if (location.includes('tokyo')) {
    return JSON.stringify({location, temperature: '10', unit: 'celsius'});
  } else if (location.includes('san francisco')) {
    return JSON.stringify({location, temperature: '72', unit: 'fahrenheit'});
  } else {
    return JSON.stringify({location, temperature: '22', unit: 'celsius'});
  }
}
```
