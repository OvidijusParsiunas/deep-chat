# <img src={claudeLogo} width="38" style={{float: 'left', marginTop: '10px', marginRight: '10px'}} /><span className="direct-service-title">Claude</span>

Properties used to connect to [Claude](https://www.anthropic.com/claude).

### `claude` {#claude}

- Type: `true` | \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `model?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `max_tokens?: number`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `stop_sequences?: string[]`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `system_prompt?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`tools?: ClaudeTool[]`](#ClaudeTool), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`tool_choice?: ClaudeToolChoice`](#ClaudeToolChoice), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`function_handler?: FunctionHandler`](#FunctionHandler), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`mcp_servers?: ClaudeMCPServer[]`](#ClaudeMCPServer), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `metadata?: {user_id?: string}`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; [`thinking?: ClaudeThinking`](#ClaudeThinking), <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `service_tier?: "auto" | "standard_only"`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `inference_geo?: string` <br />
  \}
- Default: _\{model: "claude-sonnet-4-6", max_tokens: 4096\}_

Connect to Claude's [Messages API](https://docs.anthropic.com/en/api/messages). You can set this property to _true_ or configure it using an object:<br />
`model` is the Claude model to use (e.g., "claude-sonnet-4-6", "claude-3-haiku-20240307"). <br />
`max_tokens` is the maximum number of tokens to generate. <br />
`stop_sequences` defines sequences where the API will stop generating. <br />
`system_prompt` provides behavioral context and instructions to the model. <br />
[`tools`](#ClaudeTool) is an array that defines functions that the model can call. <br />
[`tool_choice`](#ClaudeToolChoice) controls which (if any) tool should be called. <br />
[`function_handler`](#FunctionHandler) is the actual function called with the model's instructions. <br />
`mcp_servers` enables integration with [Model Context Protocol](https://docs.anthropic.com/en/docs/agents-and-tools/mcp-connector) servers. <br />
`metadata` is an object describing the request, currently supporting a `user_id` string for [abuse detection](https://docs.anthropic.com/en/api/messages#body-metadata). <br />
[`thinking`](#ClaudeThinking) enables [extended thinking](https://docs.anthropic.com/en/docs/build-with-claude/extended-thinking) where the model reasons through a problem before responding. <br />
`service_tier` selects the [request priority](https://docs.anthropic.com/en/api/service-tiers) — `"auto"` uses standard with priority fallback, `"standard_only"` forces standard tier only. <br />
`inference_geo` is the geographic region for inference processing. <br />

## Tool Calling

Claude supports [tool calling](https://docs.anthropic.com/en/docs/tool-use) functionality:

### `ClaudeTool` {#ClaudeTool}

- Type: \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `name: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `description: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `input_schema:` \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `type: "object"`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `properties: object`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `required?: string[]` <br />
  &nbsp;&nbsp;&nbsp;&nbsp; \}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `input_examples?: object[]`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `defer_loading?: boolean`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `type?: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `allowed_callers?: string[]` <br />
  \}

Array describing tools that the model may call. <br />
`name` is the name of the tool function. <br />
`description` explains what the tool does and when it should be used. <br />
`input_schema` defines the parameters the tool accepts in JSON Schema format. <br />
`input_examples` is an array of example input objects that demonstrate valid tool calls to the model. Each example must conform to `input_schema`. See [Providing tool use examples](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/define-tools#providing-tool-use-examples). <br />
[Advanced tool use](https://www.anthropic.com/engineering/advanced-tool-use): <br />
`defer_loading` does not load the tool in the initial context so it can be discovered dynamically. <br />
`type` defines the tool type. <br />
`allowed_callers` are programmtic execution tools that can call this tool. <br />

### `ClaudeMCPServer` {#ClaudeMCPServer}

- Type: \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `type: "url"`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `url: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `name: string`, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `authorization_token?: string` <br />
  \}

Configuration for [Model Context Protocol](https://docs.anthropic.com/en/docs/agents-and-tools/mcp-connector) server integration. <br />
`type` must be "url" for URL-based MCP servers. <br />
`url` is the endpoint URL of the MCP server. <br />
`name` is a unique identifier for the MCP server. <br />
`authorization_token` is an optional token for server authentication. <br />

### `ClaudeThinking` {#ClaudeThinking}

- Type: \{`type: "enabled"`, `budget_tokens: number`, `display?: "summarized" | "omitted"`\} | \{`type: "disabled"`\} | \{`type: "adaptive"`, `display?: "summarized" | "omitted"`\}

Configuration for [extended thinking](https://docs.anthropic.com/en/docs/build-with-claude/extended-thinking). <br />
When `type` is `"enabled"`, `budget_tokens` sets the maximum number of tokens the model can spend on internal reasoning (minimum 1024). <br />
`type: "adaptive"` lets the model decide how much to think based on the request. <br />
`display` controls whether reasoning content is `"summarized"` or `"omitted"` from the response. <br />

### `ClaudeOutputConfig` {#ClaudeOutputConfig}

- Type: \{<br />
  &nbsp;&nbsp;&nbsp;&nbsp; `format:` \{`type: "json_schema"`, `schema: object`\}, <br />
  &nbsp;&nbsp;&nbsp;&nbsp; `effort?: "low" | "medium" | "high" | "xhigh" | "max"` <br />
  \}

Constrains model output to a specific format. <br />
`format.type` is currently `"json_schema"`, and `format.schema` is a [JSON Schema](https://json-schema.org/) describing the response shape. <br />
`effort` is an optional hint for how much compute the model should spend on producing schema-conformant output. <br />

### `FunctionHandler` {#FunctionHandler}

- Type: ([`functionsDetails: FunctionsDetails`](/docs/directConnection#FunctionsDetails)) => `{response: string}[]` | `{text: string}`

The actual function that the component will call if the model wants to use tools. <br />
[`functionsDetails`](/docs/directConnection#FunctionsDetails) contains information about what tool functions should be called. <br />
This function should either return an array of JSONs containing a `response` property for each tool function (in the same order as in
[`functionsDetails`](/docs/directConnection#FunctionsDetails)) which will feed it back into the model to finalize a response, or return a JSON containing
`text` which will immediately display it in the chat.

```js
          },
          required: ['location'],
        },
      },
    ],
    function_handler: (functionsDetails) => {
      return functionsDetails.map((functionDetails) => {
        return {
          response: getCurrentWeather(functionDetails.arguments),
        };
      });
    },
    key: 'placeholder-key',
  },
};
```

```js
chatElementRef.directConnection = {
  claude: {
    tools: [
      {
        name: 'get_weather',
        description: 'Get the current weather in a given location',
        input_schema: {
          type: 'object',
          properties: {
            location: {
              type: 'string',
              description: 'The city and state, e.g. San Francisco, CA',
            },
            unit: {type: 'string', enum: ['celsius', 'fahrenheit']},
          },
          required: ['location'],
        },
      },
    ],
    function_handler: (functionsDetails) => {
      return functionsDetails.map((functionDetails) => {
        return {
          response: getCurrentWeather(functionDetails.arguments),
        };
      });
    },
    key: 'placeholder-key',
  },
};

function getCurrentWeather(location) {
  location = location.toLowerCase();
  if (location.includes('tokyo')) {
    return JSON.stringify({location, temperature: '10', unit: 'celsius'});
  } else if (location.includes('san francisco')) {
    return JSON.stringify({location, temperature: '72', unit: 'fahrenheit'});
  } else {
    return JSON.stringify({location, temperature: '22', unit: 'celsius'});
  }
}
```

## Types

Types shared by Claude properties:

### `ClaudeToolChoice` {#ClaudeToolChoice}

- Type: \{`type: "auto" | "any"`, `disable_parallel_tool_use?: boolean`\} | \{`type: "tool"`, `name: string`, `disable_parallel_tool_use?: boolean`\} | \{`type: "none"`\}

Controls which (if any) tool should be called. <br />
`"auto"` lets the model decide whether to use a tool. <br />
`"any"` forces the model to use one of the provided tools, but doesn't force a particular one. <br />
`"tool"` forces the model to always use the tool with the specified `name`. <br />
`"none"` prevents the model from using any tools. <br />
`disable_parallel_tool_use` limits the model to a single tool call per response. <br />
